# 账号生命周期

这页说明一个账号从“配置好”到“正常运行”的完整生命周期。

## 一、准备阶段

账号需要具备：

- `*_config.json`
- `state.json`
- `presets.json`
- `.session`

## 二、启动阶段

启动时会依次完成：

1. 读取账号配置
2. 检查管理员入口配置
3. 连接 Telegram
4. 刷新账户余额快照
5. 执行挂单自愈
6. 保存状态
7. 发送“脚本启动成功”通知

## 三、运行阶段

运行中主要会循环处理：

- 盘口触发
- 模型判断
- 按钮点击
- 结算回写
- 风控判断
- 状态刷新

## 四、暂停与恢复

常见暂停原因：

- 手动暂停
- 连输保护
- 盈利达标
- 炸号保护
- 资金不足
- 模型异常

恢复方式：

- 手动 `resume`
- 自动倒计时恢复
- 模型恢复后自动继续运行

## 五、更新与重启

更新后，一般需要：

1. `update`
2. `restart`
3. `status`

重启成功后，当前版本会主动发一条启动成功通知。
