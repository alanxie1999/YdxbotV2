# 固定数据规律下注功能

## 功能概述

当检测到特定的 6 位固定数据序列时，系统会按照预设的规律自动下注，而不是下相反的数。

## 支持的固定模式

| 序列模式 | 模式名称 | 规律说明 | 下注策略 |
|---------|---------|---------|---------|
| `101010` | 交替循环 | 大小交替出现 | 按照交替规律继续下注（下最新一手的同向） |
| `111111` | 大龙延续 | 连续 6 次大 | 继续下大（1） |
| `000000` | 小龙延续 | 连续 6 次小 | 继续下小（0） |

## 实现原理

### 1. 常量定义

在 `zq_multiuser.py` 中定义以下常量：

```python
# 固定数据规律：检测到特定 6 位序列后，按照规律下注（不下相反的数）
FIXED_PATTERN_TRIGGER_WINDOW = 6
FIXED_PATTERNS = {
    "101010": {"follow": "101010", "label": "交替循环"},
    "111111": {"follow": "1", "label": "大龙延续"},
    "000000": {"follow": "0", "label": "小龙延续"},
}
```

### 2. 检测函数

`_detect_fixed_pattern_signal(history, window)`:
- 检查最近 6 手的历史序列
- 如果匹配到固定模式，返回对应的规律信息
- 否则返回 `{"active": False}`

### 3. 应用覆盖

`_apply_fixed_pattern_override(rt, history, prediction)`:
- 在模型预测之后、实际下注之前调用
- 如果检测到固定模式，强制覆盖预测结果
- 更新运行态信息（`rt`），记录固定规律的触发状态

### 4. 清除运行态

`_clear_fixed_pattern_runtime(rt)`:
- 在需要时清除固定规律的运行态标记
- 与 alternation_break 一起管理

## 下注逻辑

### 101010（交替循环）

当检测到 `101010` 模式时：
- 最新一手是 `0`（小），则继续下 `0`（小）
- 最新一手是 `1`（大），则继续下 `1`（大）
- **不会下相反的数**

示例：
```
历史：[..., 1, 0, 1, 0, 1, 0]  (从远到近)
检测：101010 交替循环
下注：0（小）  # 最新一手是 0，继续跟 0
```

### 111111（大龙延续）

当检测到 `111111` 模式时：
- 继续下 `1`（大）
- 相信长龙会延续

示例：
```
历史：[..., 1, 1, 1, 1, 1, 1]  (从远到近)
检测：111111 大龙延续
下注：1（大）  # 继续跟大
```

### 000000（小龙延续）

当检测到 `000000` 模式时：
- 继续下 `0`（小）
- 相信长龙会延续

示例：
```
历史：[..., 0, 0, 0, 0, 0, 0]  (从远到近)
检测：000000 小龙延续
下注：0（小）  # 继续跟小
```

## 触发优先级

```
模型预测 → 交替突破覆盖 → 固定规律覆盖 → 实际下注
```

固定规律覆盖在交替突破覆盖之后执行，确保多种信号可以同时生效。

## 运行态信息

当固定规律触发时，会在运行态（`rt`）中记录以下信息：

- `fixed_pattern_active`: 固定规律是否激活
- `fixed_pattern_seq`: 检测到的序列（如 "101010"）
- `fixed_pattern_side`: 下注方向（"大"或"小"）
- `fixed_pattern_label`: 模式标签（如"交替循环"）
- `last_predict_source`: 预测来源（"fixed_pattern"）
- `last_predict_tag`: 预测标签（"FIXED_PATTERN"）
- `last_predict_confidence`: 置信度（100）
- `last_predict_reason`: 预测原因说明

## 配置说明

当前功能为内置功能，无需额外配置。

如需修改触发窗口大小，可调整 `FIXED_PATTERN_TRIGGER_WINDOW` 常量（默认 6）。

如需添加更多固定模式，可在 `FIXED_PATTERNS` 字典中添加条目：

```python
FIXED_PATTERNS = {
    "101010": {"follow": "101010", "label": "交替循环"},
    "111111": {"follow": "1", "label": "大龙延续"},
    "000000": {"follow": "0", "label": "小龙延续"},
    # 添加新模式的示例：
    # "110011": {"follow": "1", "label": "对称模式"},
}
```

## 注意事项

1. **不下相反的数**: 功能设计原则是"顺势而为"，不会下与规律相反的数
2. **覆盖模型预测**: 固定规律的优先级高于模型预测，即使是 SKIP 也会被覆盖
3. **连续触发**: 如果每一手都符合固定模式，会连续触发下注
4. **与其他功能共存**: 与风控、暂停等功能正常交互，不会突破风控限制
