# 命令详解

这页按日常使用频率整理命令，并补上：

- 功能说明
- 典型界面示意
- 操作技巧

说明：

- 如果管理员入口使用的是 `telegram_id`，直接发送命令即可
- 如果管理员入口使用的是 `telegram_bot`，通常使用 `/命令` 形式

## 一、`st [预设名]`

### 功能说明

启动并切换到指定预设。

作用：

- 把当前账号切到新的预设参数
- 进入可下注状态
- 同时自动触发一次测算报告

### 常见写法

```text
st 5k
st 10w
```

### 典型界面示意

```text
🎯 预设启动成功: 5k

当前账号已经切换到新的预设，后续将按这套参数进入可下注状态。
策略参数：1 12 3.0 2.5 2.2 2.1 5000
```

### 操作技巧

- 这是最常用的入口命令之一
- 每次切换预设后，建议紧接着看一次 `status`
- 如果要先看资金链能不能覆盖，再执行 `yc 5k`

## 二、`status`

### 功能说明

查看当前账号完整状态面板。

它会集中展示：

- 当前运行状态
- 当前预设
- 下一手计划下注
- 账户余额 / 菠菜资金
- 模型状态
- 本轮目标与本轮损益
- 最近结果

### 常见写法

```text
status
```

### 典型界面示意

```text
【 状态监控 】 🟢 运行中
更新：03-31 18:20:11
版本：v1.2.5(4dc18d1)
方案：5w

📦 新版本：v1.3.0（可更新）

🤖 模型状态：🟢 正常
当前模型：qwen/qwen3-next-80b-a3b-instruct
最近成功：18:19:58

🎯 即时下注
├ 下一预测：大
├ 计划下注：14.15 万
├ 单轮目标：100.00 万
├ 本轮损益：+18.13 万 📈
└ 目标进度：[▓░░░░░░░░░] 18.1%
```

### 操作技巧

- 日常最常用的查看命令
- 出现异常、暂停、连续兜底、点击失败后，第一时间先看它
- 更新成功但还没重启时，也能在顶部看到新版本提示

## 三、`pause` / `resume`

### 功能说明

- `pause`：手动暂停当前账号押注
- `resume`：恢复当前账号押注

### 常见写法

```text
pause
resume
```

### 典型界面示意

```text
⏸️ 已暂停当前账号押注

当前账号后续不会自动下注，已有状态会被保留。
```

```text
▶️ 已恢复当前账号押注

后续会继续等待有效盘口触发，不会立即补发历史下注。
```

### 操作技巧

- 需要人工介入时先 `pause`
- 不要在连输高压位反复切换
- 恢复后建议再看一次 `status`

## 四、`balance`

### 功能说明

刷新当前账户余额，并同时显示当前菠菜资金。

### 常见写法

```text
balance
```

### 典型界面示意

```text
💰 账户余额查询成功

余额已刷新到当前最新值。
账户余额：123.40 万
菠菜资金：45.60 万
```

### 操作技巧

- `gf` 修改资金前后，建议都执行一次
- 如果你怀疑余额不同步，也先看这条

## 五、`gf [金额]`

### 功能说明

设置当前账号的菠菜资金。

### 常见写法

```text
gf 2000000
```

### 典型界面示意

```text
✅ 菠菜资金已更新

新的菠菜资金已经写入当前账号状态。
当前金额：200.00 万
```

### 操作技巧

- 输入单位是原始整数，不是“万”
- 如果输入超过账户余额，系统会自动压到可用上限
- `gf` 是菠菜资金和账户余额唯一允许发生关联的命令

## 六、`stf [数字]`

### 功能说明

设置本轮目标金额。

单位直接用“万”。

### 常见写法

```text
stf 100
```

含义：

- 本轮目标金额 = `100 万`

### 典型界面示意

```text
✅ 本轮目标金额已更新

当前目标：100.00 万
```

### 操作技巧

- 这是当前替代旧 `set` 的唯一目标金额命令
- 不再通过一个命令同时改炸号、暂停和盈停参数

## 七、`wlc [数字]`

### 功能说明

设置连输告警阈值。

### 常见写法

```text
wlc 3
wlc 5
```

### 典型界面示意

```text
✅ 连输告警阈值已更新

后续达到该连输次数时，系统会发出高优提醒。
当前阈值：3 次
```

### 操作技巧

- 阈值越低，提醒越早
- 阈值不是止损，只是提醒

## 八、`yss`

### 功能说明

查看当前账号可用的全部预设。

### 常见写法

```text
yss
```

### 典型界面示意

```text
📚 当前预设列表

'5k ': ['1', '12', '3.0', '2.5', '2.2', '2.1', '5000']
'1w ': ['1', '11', '3.0', '2.5', '2.2', '2.1', '10000']
'2w ': ['1', '10', '3.0', '2.5', '2.2', '2.1', '20000']
...
```

### 操作技巧

- 先看 `yss`，再决定 `st 5k` 或 `st 10w`
- 如果你怀疑预设有重复或旧预设回流，也先看这条

## 九、`ys [名称] ...`

### 功能说明

新增或覆盖一个预设。

参数顺序：

```text
ys [名称] [连续] [止损] [一输] [二输] [三输] [四输] [首注]
```

### 常见写法

```text
ys 2w 1 10 3.0 2.5 2.2 2.1 20000
```

### 操作技巧

- 新预设名建议沿用 bankroll 风格
- 保存后建议马上执行一次 `yss`

## 十、`yc`

### 功能说明

测算预设或临时参数的资金链。

### 常见写法

```text
yc 5k
yc 1 12 3.0 2.5 2.2 2.1 5000
```

### 典型界面示意

```text
🔮 已根据当前预设自动测算

策略参数
预设名称：5k
菠菜资金：300.00万
...
连数|倍率|下注|盈利|所需本金
```

### 操作技巧

- 切预设前先测算最稳
- 重点看“最多能覆盖到几连”

## 十一、`stats`

### 功能说明

查看当前策略链的连大、连小、连输统计。

### 常见写法

```text
stats
```

### 典型界面示意

```text
📊 统计概览

说明：仅统计当前策略链中已结算记录；未结算和异常挂单不计入。

<pre>最近局数“连大、连小、连输”统计
...
</pre>
```

### 操作技巧

- 用它看当前链路节奏
- 不要把它理解成“所有历史原始日志”的机械统计

## 十二、`res tj / res state / res bet`

### 功能说明

- `res tj`：重置收益/胜率统计
- `res state`：重置运行状态和历史
- `res bet`：只重置当前倍投链

### 常见写法

```text
res tj
res state
res bet
```

### 操作技巧

- 删 `state.json` 之前，先想想是不是 `res state` 就够了
- `res bet` 更适合只想清当前链，不想清所有历史的时候

## 十三、`ver / update / reback / restart`

### 功能说明

- `ver`：查看当前版本和可更新信息
- `update [版本]`：更新到指定 tag 或最新版本
- `reback [版本]`：回退到指定版本
- `restart`：重启脚本

### 常见写法

```text
ver
update v1.2.5
reback v1.2.4
restart
```

### 典型界面示意

```text
✅ 更新成功

代码已切到目标版本。
目标版本：v1.2.5
当前版本：v1.2.5
重启命令：`restart`
```

```text
✅ 脚本启动成功
重启日期：2026 年 3 月 31 日 14:40
脚本版本：v1.2.5(4dc18d1)
账户余额：1471.20 万
菠菜资金：946.14 万
```

### 操作技巧

- 更新后一定要 `restart`
- 如果 `update` 被阻塞，先看排障页
