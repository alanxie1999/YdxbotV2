# 投注与结算流程

这页说明脚本从收到盘口消息到结算回写的主流程。

## 一、投注前

脚本会依次做这些事：

1. 检查当前是否允许下注
2. 检查资金是否足够
3. 调模型或统计兜底判断方向
4. 计算下一手金额
5. 组合按钮并执行点击

## 二、投注执行

如果点击成功：

- 记入下注链
- 等待结算
- 刷新状态面板

如果点击失败：

- 不再错误推进倍投链
- 会明确提示：
  - 响应超时
  - 窗口失效
  - 其他执行失败

## 三、结算回写

收到结算消息后，脚本会：

1. 识别开奖结果
2. 回写本手输赢
3. 更新余额、收益、连输、连续押注
4. 必要时触发：
   - 连输告警
   - 连输终止
   - 盈利暂停
   - 炸号暂停

## 四、链路保护

当前版本已经加强这些保护：

- 挂单自愈
- 点击失败不推进金额
- 高金额多按钮点击超时放宽
- 连输与回补链路按真实已结算记录回算
