# 统计口径说明

这页专门解释 `stats`、连输告警和回补结束消息里的统计口径。

## 一、stats 统计什么

`stats` 主要统计三类内容：

- 连大
- 连小
- 连输

## 二、当前 stats 的口径

当前版本已经收紧为：

- 只统计当前策略链中的数据
- 连输只看已结算记录
- 未结算与异常挂单不计入

## 三、为什么会和肉眼复盘不完全一样

因为脚本会明确排除这些数据：

- 未结算押注
- 异常未结算挂单
- 非当前策略链的旧残留

这样做的目的，是让 `stats` 更接近当前真实运行态，而不是混入脏数据。

## 四、表头为什么不是固定 1000 / 500 / 200 / 100

当前版本会根据真实历史长度动态显示窗口大小。

例如：

- 历史只有 `168` 条，就显示 `168`
- 不再强行写 `1000`

## 五、连输已终止里的“本轮净盈利”

当前版本已经改成：

- 按整轮回补链中所有已结算 `profit` 求和

不再依赖：

- `gambling_fund` 差值

这样更稳，也更符合真实净收益。
