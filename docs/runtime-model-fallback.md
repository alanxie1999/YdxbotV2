# 模型兜底与恢复机制

这页说明模型链在运行中的基本行为。

## 一、正常链路

正常情况下，脚本每次下注前会：

1. 读取当前模型
2. 按 fallback chain 调用模型
3. 得到方向、理由和置信度
4. 再进入下注执行

## 二、什么时候会走兜底

常见触发条件：

- 模型超时
- 模型返回无效结果
- 整条模型链都失败

这时脚本会改用统计兜底，避免整轮完全停住。

## 三、连续兜底保护

当前版本中：

- 连续兜底达到阈值后，会自动暂停
- 暂停期间后台会轮询探测模型是否恢复
- 模型恢复后，会自动恢复运行

## 四、管理员可见反馈

管理员可通过状态区或通知看到这些模型状态：

- 正常
- 已切换
- 恢复探测中
- 不可用（已自动暂停）
- 已恢复

## 五、要点

- 模型故障和策略观望是两套不同机制
- 连续观望不会直接等同于模型故障
- 高金额多按钮点击失败不会再错误推进倍投链
