# 自动化版本更新指南

## 快速更新（推荐）

每次提交代码时，一键完成版本更新、提交、打 tag、推送：

```bash
./release.sh patch "修复了 xxx 问题"
```

示例：
```bash
./release.sh patch "修复交替检测逻辑"
./release.sh minor "新增止损暂停功能"
./release.sh major "重大架构升级"
```

**自动完成**：
1. ✅ 更新版本号（patch/minor/major）
2. ✅ 更新日期到当天
3. ✅ 提交代码
4. ✅ 创建 git tag
5. ✅ 推送到远程

**之后在 Telegram 中更新**：
```text
update v2.4.4
restart
```

## 手动分步更新

### 方式 1：使用更新脚本

```bash
# 仅更新版本号和日期（不提交）
./update_version.sh patch

# 查看改动
git diff

# 手动提交和推送
git add -A
git commit -m "chore: 更新版本"
git push
```

### 方式 2：Git Hook 自动更新（高级）

安装 hook：
```bash
cp .git-hooks/post-version-update .git/hooks/post-commit
chmod +x .git/hooks/post-commit
```

**效果**：每次 `git commit` 后自动更新日期到当天

## 版本号规则

遵循语义化版本（Semantic Versioning）：

| 类型 | 格式 | 说明 | 示例 |
|------|------|------|------|
| patch | `X.Y.Z+1` | 修复 bug | 2.4.3 → 2.4.4 |
| minor | `X.Y+1.0` | 新增功能 | 2.4.3 → 2.5.0 |
| major | `X+1.0.0` | 重大变更 | 2.4.3 → 3.0.0 |

## 常见问题

### Q: 忘记打 tag 怎么办？
A: 手动补打：
```bash
git tag -a "v2.4.4" -m "Release v2.4.4"
git push origin v2.4.4
```

### Q: 如何回退版本？
A: 使用 Telegram 命令：
```text
reback v2.4.3
restart
```

### Q: 如何查看可更新版本？
A: 在 Telegram 中：
```text
ver
```

## 自动化建议

### 方案 1：每次提交自动更新 patch 版本

修改 `.git/hooks/post-commit`：
```bash
#!/bin/bash
cd /workspace
./update_version.sh patch
git add zq_multiuser.py
git commit --amend --no-edit
```

### 方案 2：定时检查版本

添加 crontab：
```bash
# 每天凌晨检查新版本
0 0 * * * cd /workspace && git fetch && git status
```

### 方案 3：CI/CD 自动发布

使用 GitHub Actions，push 到 main 分支后自动：
1. 更新版本号
2. 构建
3. 测试
4. 创建 release

## 最佳实践

1. **小步快跑**：每次改动都更新 patch 版本
2. **功能聚合**：多个相关改动累积到 minor 版本
3. **重大变更**：breaking change 更新 major 版本
4. **及时推送**：发布后立即在 Telegram 更新
5. **记录变更**：在 CHANGELOG.md 中记录更新内容
